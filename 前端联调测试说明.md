# DOC90 前端联调测试说明

本文面向**前端同学**：在本机把**依赖服务（Docker）**与 **Java 后端**跑起来后，用静态页 `frontend/combined_final.html` 联调。  
仓库根目录另有面向 AI 端的后端说明：`RUN_BACKEND_FOR_AI.md`。

---

## 1. 当前 Docker 配置说明（你需要知道的）

| 项目 | 说明 |
|------|------|
| **已有文件** | 仓库根目录 `docker-compose.dev.yml` |
| **包含服务** | MySQL 8.3、Redis 7.2、RabbitMQ 3.13（带管理插件） |
| **未包含** | **Spring Boot 后端没有 Docker 镜像**，需本机用 **JDK + Maven** 启动（见下文）。 |
| **数据库初始化** | 若存在 `db/mydatabase_dump.sql`，MySQL **首次**建卷时会自动导入；已有数据卷则不会重复执行。 |

若你希望「一条命令连后端一起容器化」，需要额外编写 `Dockerfile` 与扩展 `docker-compose`，**当前仓库未提供**。

## 2. 环境准备

- **Docker Desktop**（Windows/macOS）或 Docker Engine（Linux）
- **JDK 21**、**Maven 3.9+**（启动后端）
- 浏览器：**Chrome 或 Edge**（下载/另存为、部分 API 行为更一致）
- （可选）**Node.js**：若用 `npx serve` 起静态资源；不用也可直接用 VS Code Live Server 等

## 3. 启动依赖（MySQL / Redis / RabbitMQ）

在仓库 **`doc-fusion` 根目录**（与 `pom.xml` 同级）执行：

```bash
docker compose -f docker-compose.dev.yml up -d
```

启动后默认端口：

| 服务 | 地址 | 说明 |
|------|------|------|
| MySQL | `127.0.0.1:3306` | 用户 `root`，密码 `1234`，库名 `mydatabase` |
| Redis | `127.0.0.1:6379` | — |
| RabbitMQ | `127.0.0.1:5672` | 管理台：<http://127.0.0.1:15672>，账号密码 **`guest` / `guest`** |

**若 MySQL 初始化脚本未执行**（例如以前起过容器、卷里已有数据），需要清空卷重来：

```bash
docker compose -f docker-compose.dev.yml down -v
docker compose -f docker-compose.dev.yml up -d
```

若仓库里**没有** `db/mydatabase_dump.sql`，请用 MySQL 客户端执行 `src/main/resources/schema.sql` 建表（具体以仓库为准）。

---

## 4. 启动 Java 后端（doc-fusion）

`application-dev.yml` 里 **`doc.rabbitmq` 已默认为 `guest/guest`**，与 `docker-compose.dev.yml` 里 RabbitMQ 容器一致，**只启用 `dev` 即可**。

在项目根目录执行：

```bash
mvn -DskipTests spring-boot:run
```

（若你改过 MQ 账号，可再加 profile **`docker`** 强制对齐：`mvn ... -Dspring-boot.run.profiles=dev,docker`，见 `application-docker.yml`。）

- **本机默认端口**：`http://127.0.0.1:8081`（见 `application-dev.yml` 中 `server.port`；若被占用请改配置或停占用进程）。
- Swagger（若已启用）：`http://127.0.0.1:8081/swagger-ui/index.html`（以实际为准）。

**PowerShell 覆盖 MQ 账号**（一般不需要；本机 MQ 非 guest 时用）：

```powershell
$env:DOC_RABBITMQ_USERNAME="guest"
$env:DOC_RABBITMQ_PASSWORD="guest"
mvn -DskipTests spring-boot:run
```

## 5. 启动前端静态页（DOC90）

本目录结构示例：

```text
DOC90/
  frontend/
    combined_final.html
    img/
    ...
```

在 **`DOC90/frontend`** 下起本地静态服务，例如：

```bash
cd DOC90/frontend
npx --yes serve . -p 5173
```

浏览器访问：

```text
http://localhost:5173/combined_final.html
```

### 5.1 与后端地址对齐（必查）

打开 `combined_final.html`，搜索 **`baseUrl`**，应指向你本机后端，例如：

```javascript
const baseUrl = 'http://127.0.0.1:8081';
```

若后端端口不是 `8081`，请改成一致后**硬刷新**（Ctrl+F5）。

## 6. 不启动 Python AI 时的快速联调（推荐先跑通页面）

后端 `application-dev.yml`（或 `application.yml`）中：

```yaml
ai:
  mock:
    enabled: true
```

含义简述：

- **填表链路**：不请求 Python 的 `/api/tasks/*`，会把模板或输入文件复制到结果目录，便于测 **SSE、任务状态、下载**。
- **抽取链路**（若有）：使用配置的 mock JSON，不请求 `/analyze`。

测完后联真实 AI 时务必将 **`enabled: false`**。

## 7. 完整链路（含 Python AI，可选）

若需与 **Respond-in-90-seconds**（FastAPI，默认 `8000`）联调：

1. 按该项目 README 安装依赖并启动，例如：  
   `uvicorn api_server:app --host 0.0.0.0 --port 8000`
2. 后端 `ai.base-url` 指向 `http://localhost:8000`，且 **`ai.mock.enabled: false`**。

详见仓库根目录 `RUN_BACKEND_FOR_AI.md`。

## 8. 联调检查清单

- [ ] `docker compose -f docker-compose.dev.yml ps` 三个容器均为 Up  
- [ ] 后端日志无 RabbitMQ / MySQL 连接失败（MQ 默认已与 Docker 中 `guest/guest` 一致）  
- [ ] 浏览器访问 `baseUrl` 对应端口可达（可先试 Swagger 或健康检查）  
- [ ] `combined_final.html` 中 `baseUrl` 与后端端口一致  
- [ ] 首次测试可开 **`ai.mock.enabled: true`**，确认流程与下载再关 mock

## 9. 常见问题

**Q：前端报跨域 / CORS？**  
A：以后端实际 `SecurityConfig` 为准；若仅本地 file:// 打开 HTML，可能受限，请用 **http 静态服务**（如上一节的 `serve`）访问。

**Q：填表一直排队或消费不到？**  
A：确认 `application-dev.yml` 中 `doc.rabbitmq` 与容器一致（默认 `guest/guest`）；若本机 MQ 自建了别的用户，用环境变量或 `dev,docker` 覆盖。确认队列名与后端监听一致。

**Q：Docker 里有没有前端镜像？**  
A：**没有**。前端目前是静态资源，本地起一个静态服务器即可。
