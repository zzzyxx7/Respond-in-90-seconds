# 错误码说明

项目统一响应体格式：`{ code, message, data }`。

## 约定语义

- `200`：成功。
- `400`：请求参数或请求体错误（字段缺失、类型不匹配、校验失败）。
- `401`：未认证或 token 无效/过期。
- `403`：已认证但无权限访问资源（如非 ADMIN 访问 `/api/dev/**`）。
- `429`：请求过于频繁（限流触发）。
- `500`：系统内部异常。

## 常见场景

- 登录 token 缺失或错误：`401`，`message` 如“token 无效 / token 已过期”。
- JSON 字段类型错（例如把 `templateId` 传成对象）：`400`，`message` 提示“字段类型不匹配”。
- 访问他人任务/模板/文档集：`403` 或业务拒绝提示。
- 提交任务频率过高：`429`。
- 后端未预期异常：`500`。

## 前端建议

- `400`：提示用户修改输入。
- `401`：引导重新登录。
- `403`：提示无权限，隐藏不允许入口。
- `429`：显示“操作过于频繁，请稍后再试”并禁用按钮短时间。
- `500`：提示“系统繁忙，请稍后重试”。
